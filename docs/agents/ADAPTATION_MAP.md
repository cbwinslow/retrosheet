# Adaptation Map

This document maps the existing project assets to the four‑plane architecture
described in `docs/OVERALL_STRATEGY.md` (Data, Feature, Model, Execution).

| Plane | Existing assets (examples) | Intended role |
|-------|----------------------------|--------------|
| **Data** | `sql/020_plate_appearances.sql`, `sql/144_live_plate_appearances.sql`, `scripts/fetch_mlb_schedule.py`, `scripts/ingest_live_games.py` | Raw historical and live event ingestion, stored in `raw_retrosheet` and `raw_mlb` schemas. |
| **Feature** | `sql/050_feature_marts.sql`, `sql/060_advanced_feature_marts.sql`, `sql/122_live_pa_feature_parity.sql` | Materialized views that expose static priors, rolling historical features, and live‑state feature parity. |
| **Model** | `scripts/train_models.py`, `scripts/promote_best_models.py`, `scripts/fast_prediction_service.py` | Training pipelines, model registry (currently informal), and inference service. |
| **Execution** | `baseball-chatbot-ui/app/api/predict/route.ts`, `baseball-chatbot-ui/app/api/live-odds/route.ts` | API endpoints that serve model predictions and expose live odds; future pricing and risk layers will live under `services/`. |

The map is intentionally high‑level; detailed column‑level mappings are left to the
implementation of wrapper views and helper functions (see Phase 1 tasks).
